基于hexo+GitHub Pages建立静态博客的泛用型指南

起

前几年玩过 WordPress 建站，其实还蛮好玩的，也正是因此为契机才正式接触到了编程。整了一年的服务器最后发现一年下来也就写了十来篇，也就懒得续费了。时值 HTML 5 的规范刚定了下来，微信公众号在乱军中杀出一条血路。好像 hexo 与 GitHubpages 也是当年出来的，总而言之当年服务器懒得续费之后便把文章转向了 GitHubpages 和微信公众号两边。

当年的微信公众号才起步，网页编辑器甚至可以改 HTML 源码，而对应的 GitHub pages 这边就很麻烦，要先写好文章，再在本地生成文件，再提交到服务器。因为我一开始便是接触的 WordPress 这种可以直接在网页上编辑的动态博客，对于静态博客这种东西还是不那么喜欢。渐渐地我就没去管静态博客了。

2020年初，中国遭遇到了名为 2019-nCoV 的病毒所威胁，这个事件待疫情结束后我再看看记录不记录吧。总而言之，[2020/2/7] 我在微信公众号上久违地更新了一篇《小学生观察笔记》，然后在阅读只有 2 的情况下因为“遭到多人举报而删除”了。毕竟大公司也要恰饭，思来想去我便把公众号注销了，同时开始从头维护一个博客。

本文用于记录博客的建立与维护。

常驻系统为 win 10 所以本文主要面向 Windows系统 。但我也是菜鸡一个，如果有其他的人看到了这篇文章觉得哪里不对可以教教我；个人博客，喷子大可不必，直接右上角即可关闭。

承

首先确立流程：

  1. 安装 Git
  2. 安装 Node.js
  3. 安装 hexo
  4. GitHub 创建个人仓库
  5. 生成 SSh key 并添加到 GitHub
  6. 将 hexo 部署到 GitHub
  7. 建站完成，然后开始写泛用性指南

1、安装 Git

Git 是个将本地文件更方便上传到 GitHub 去的工具，还是个很优秀的版本管理系统。以后要是有心情，就再细细去学 git 的教程。

下载地址：从官网[下载Git即可，安装选项全部默认就行了。

后文的“输入”默认都是用 git 执行

ps：可以在cmd里输入 git -v 查看版本信息确认是否安装成功。

pps：cmd是指命令提示符，开始菜单输入cmd即可打开。

2、安装 Node.js

Hexo 基于 Node.js 编写，所以需要安装 Node.js。

下载地址：同样是官网下载稳定版即可。安装同样是全部默认。

ps：在 cmd 输入 node -v 查看版本信息确认是否安装成功。

3、安装 Hexo

前期准备到此为止，这里开始进入正题。

首先在你想要的地方创建一个文件夹“小学生的秘密基地”，用于存放blog文件。

在文件夹内右键选择 ‘Git bash here’，用于定位到该目录。

输入 npm install hexo-cli -g，用于安装 Hexo。或许有报错，但没关系。

ps：输入 hexo -v 确认版本信息确认是否安装成功。

输入 hexo init Blog 初始化网站，代码中的 Blog 叫啥都行，后文默认 Blog 文件夹为根文件夹。

输入 cd blog 进入博客文件夹。

输入 npm install 安装必备组件。

至此，Blog 目录下应该生成了些许文件：

  • public：存放生成的页面
  • scaffolds：生成文章的模板
  • source：存放写的文章
  • themes：主题
  • _config.yml：博客的配置文件

可以用记事本稍微修改修改下配置文件。

输入hexo g 生成静态页面。

输入 hexo s 打开本地服务器可以本地预览，我以后基本都省略了这步。

在浏览器中打开 http://localhost:4000就可以看到博客的预览了。

Ctrl+c关闭本地服务器。

4、创建 GitHub 仓库

没注册的先注册，用户名就是将来的域名前缀。

新建项目仓库：New repository，项目名就是‘yourname.github.io’，这里的yourname 就是你的用户名，我记得不是自己的用户名那个域名就会很长，虽然可以通过绑定自己的个人域名来解决。总之问题不大。

  • 记得勾选README的初始化。

创建完成后，仓库的右侧有个 settings 按钮，下拉有个 GitHub Pages 的分栏，Source里应该有个选项，选 master branch 就会将你的网站在那个后缀为‘GitHub.io’的链接上公开。

5、生成 SSH key 并部署到 GitHub

整这玩意就是为了以后上传文件的时候不用输密码（除非你在设置 ssh key 的时候设置了密码）

首先创建一个 ssh key ：

输入：

ssh-keygen -t rsa -C "your_email@example.com"

邮箱名想改的改，不改会不会出错我没试过。

这时候会让你输入一个文件名，用于保存刚才生成的 ssh key ，我换了几个文件夹都没成功，不输入直接回车默认倒成功了。

接着会提示你输入两次密码（是ssh key 的密码，用于日后 push 文件时要输的）直接回车，以后轻松。

接下来显示什么Your public key has been saved 啥的就表示你的ssh key已经创建成功了。

添加你的 ssh key 到 GitHub 上：

输入 clip < ~/.ssh/id_rsa.pub ，用于拷贝id_rsa.pub 文件里的内容。不嫌麻烦用记事本打开复制应该也是可以的。

登录 GitHub 账号，右上角Account Setting ，选择菜单栏里的 SSH key 。

点击 Add SSH key，把刚刚复制的内容粘贴单到对应输入框（前后不要有空格），Title的输入框里随便取个名字就行。

输入 ssh -T git@github.com

会显示 Are you sure you want to continue connecting (yes/no)?

输入 yes

这时候会要你输密码。就刚刚的ssh key的密码。

总之成功会有 successfully 的字眼，失败会有 denied 字眼。

6、部署 hexo 到 GitHub

最后一步了，博客根目录输入 npm install hexo-deployer-git --save 安装 deploy-git。

修改根目录下的config文件：

然后输入 hexo clean 清除你生成的东西，有时候改了东西网页却没变可以用这个清除一下。

输入 hexo g 生成静态文件

输入 hexo d 部署到服务器

ps：deploy阶段可能会弹出要求输入密码

过一小会就可通过之前的 GitHub pages 设置的域名打开博客了。

《京紫外传》观后感

紫罗兰永恒花园外传：永远与自动手记人偶

没能找到高潮的我，准备写下关于这个电影的观后感。

（为了方便，TV版统称京紫，电影统称外传）

从我看的番剧和电影来看，日本人表达情感和温柔都是内敛的，比如夏目里的温柔就是润物细无声的，就像下雨一样，一点点的慢慢的渗入故事里的那种温柔。

京紫我没看完，但是我能感受到那种温柔，是另一种更直接的温柔。

要谈京紫里的温柔，要先谈EVA设定里的AT力场（绝对领域），在EVA中，AT力场被升华为“将自我与客观世界分割开的一堵墙”，揭示出了“每个生物都有AT力场”的道理。AT力场同时也是寂寞与痛苦的原因。在主流心理学中AT力场指当一个孤独症患者在身体或精神上感到对他的亲近时，如果接近者没有被他所认可，他将会恐慌而失去理智。

说这么多，京紫的故事就是像EVA里一样薇尔莉特打破了这层AT力场，AT力场里的人、AT力场里的故事、AT力场里的温柔就从这层壳里流淌出来。京紫就大概是这么个流程：被AT力场隔绝→打破AT力场→看到内在。而TV版的节奏把控相对较好，所以高潮往往能在打破AT力场时看到内在时恰巧的经过。

而外传这一点做的并不好。

剧情是老套的没有血缘关系的两个亲人相互思念的故事。

所谓的女主，在里面也就是个纽带，总感觉送信那小哥比薇尔莉特还更有主角的感觉。

所以主角就约克和泰勒呗。我寻思高潮应该是她两见面拥抱然后抱着转圈圈呗。

然而并没有。

电影到后半段的时候，我真的一直在期待一个高潮的到来，类似TV版里的高潮片段，我至今都觉得只要有一个高潮的出现就能让这个电影观赏性极佳。

可惜并没有等到。

纵观整个故事，姐姐是想与妹妹相见（其实一开始我感觉姐姐是想要自由，羡慕薇尔莉特能到处走），妹妹是想与姐姐相见，两人目的一致但是从结果上说都没有实现。你说两人目的都没实现的结局能算happy end么？

最后的高潮部分我就觉着很奇怪，为什么是让快递小哥带着妹妹来，那薇尔莉特和姐姐的羁绊用处是什么。让薇尔莉特来送这封信会不会更有冲击力一点。毕竟能连接成一条线：姐姐—薇尔莉特-妹妹，形成一个桥梁一个纽带。就比如姐姐给薇尔莉特织辫子，薇尔莉特给妹妹织辫子，一种循环本能为一种纽带的。

抛去故事上的一些问题，我想外传还是不错的，京都的画面令人放心，甚至令人懒得吹，人物情感的转变能很好的从画面上表现出来，妹妹的那句送去的不止是信件，而是人们的心意人们的幸福，其实主旨很不错，正如快递小哥所说：

没有信件是不需要被送达的

05 命运

我并不是一个宿命论的忠实拥趸。但今天想谈谈命运。
我看了看微博，我发现中国人大多都喜欢听“善有善报恶有恶报”这一套，但当命运不公时，便开始叫喊着“杀人放火金腰带”这套非黑即白的逻辑，简单的逻辑感觉什么时候都吃香。
总不能因为过去这一百年我们一败涂地，就放弃争取胜利吧。

因而我就思索了一下中国古代的悲剧：首先想到的是窦娥，窦娥具体是啥有点想不起来了，但结局我记得是六月飞雪，然后鬼魂出现，再重审此案，洗刷冤情；再一个是赵氏孤儿，一个被杀全家复仇成功的故事；再一个白娘子，我没看但是我听说是得道成仙的结局。

这么看中国文化应该是没有悲剧的。
要说惨不惨，那都惨，但结局都是群众喜闻乐见乐意接受的结局。从小就耳读目染“善有善报”的理论，本质上也是满足了人们那颗“正义必胜”的心理。
要谈悲剧，自然逃不开古希腊悲剧。
古希腊悲剧的一大特点便是，命运的喜怒无常。刚开始看《荷马史诗》的时候，我就很纳闷，这雅典娜不是女神么，怎么还随便杀人的。

后来渐渐的理解了古希腊这种哲学思想，雅典娜是战争女神，她便代表着命运，即使她不公，但你还真就无能为力。
在这个世界上，我除了冲着人们大笑以外，对他们无可奈何，那我干脆就加入马戏团，让自己笑个够。
亚里士多德在《诗学》中曾专门探讨悲剧的含义。他认为悲剧的目的是要引起观众对剧中人物的怜悯和对变幻无常之命运的恐惧，由此使感情得到净化。悲剧中描写的冲突往往是难以调和的，具有宿命论色彩。
很多时候，古希腊悲剧里看不到“善恶终有报”，遭受厄运也不会有补偿。这便是神意，现代话这便是命运。

而真正的悲剧恰就如此。
勇敢就是在你还没开始的时候就知道自己注定会输，但依然义无反顾地去做一个人很少能赢，但也总会有赢的时候。
雅典娜或许是想再来一场特洛伊战争，但我们看到木马时也应大喊。预言者的终焉之时，却也是人类的光。
一个对雅典娜说出“明白”，但我还是要喊出来的勇气。

For the glory of mankind!

04 浪漫

本系列纯属偏见

本着任何的观点本质上都是不同学识的人在不同的世界观下对同一件事情的偏见，所以任何的偏见都在某种意义上属于对的。以此为本系列大前提而进行的任何偏见言论，都应当被尊重。

本系列大多数为碎碎念，想到什么写什么，并且我不对其负责。

日常生活的无聊程度与渴望浪漫的程度成正相关。所谓的“浪漫”，在很大程度上是在指抛弃“日常”、打破“日常”。日常生活越无聊，越渴望浪漫。换句话说就是日常的无聊正是追求浪漫的源泉。

首先定义何为“浪漫”，狭义上的浪漫来源于19世纪20年代至19世纪末诞生于西欧的浪漫主义，这里的浪漫特指对现实世界感到不满，对未来充满期望和追求，理想远大，不甘堕落的个人英雄主义情怀。[1]
浪漫主义作为一股文学思潮，本质上就是一次对古典主义文学的反动，是对文艺复兴时期人本主义的继承和发扬。强调主观精神、个人主义和批判意识。正是如此，浪漫主义文学带来了个性解放和追求自由的浪潮。浪漫主义时期的文学创作具有显著的艺术特征，相对于传统古典文学表现出了天马行空的想象力，强调内心的想象作用，摒弃对现实世界的模仿，比如爱伦·坡的诗歌则多被认为是自我精神的投射，使某一具体实物包涵着广阔天地和深沉的感情。

看上去浪漫主义文学与现实中的浪漫并没多少关系。我也没那个本事来剖析浪漫的发展与演变，好在小学生观察笔记有胡写瞎写免责协议。

浪漫主义首先就是对现实世界是完全否定的，他们执着于追求无限的诗意的世界，就必须通过一种方式对现实的否定与超越，这种方式对浪漫主义来说又是浪漫主义式的反讽。

反讽在西方文化中最早形成于古希腊，作为辩论的修辞手法出现，我没记错的话是苏格拉底发明的这种修辞手法。到浪漫主义时期，反讽则从辩论的修辞手法过度到了哲学与美的层面，并成为浪漫主义文学创作中的一种特定叙述体式。[2]

而人类世界的荒诞也借浪漫主义之手展现在世人面前。

反讽的力量在于对荒诞的现实说不，作为一种行动哲学，赋予反讽以批判、反抗、否定现实的力量。而浪漫主义的反讽则是在‘诗意的栖居’与‘生存哲学’中将现实与理想统一。

写到这，大致定义了什么是浪漫主义，所谓的浪漫主义是在有限的生命里寻求无限的可能，在明确了世界的矛盾性和荒谬性之后渴望超脱于现实之上，在反讽中消灭有限性和客观性，使世界浪漫化，从而通向无限和自由，最终到达“诗意的栖居”。

而现世中的浪漫，经过几个世纪的车轮碾压，或许有了新的定义，但本质上不会变的，浪漫主义实质上就是个人追求自由的过程，是追求个人精神的自由，是对现实的否定，是在无限的渴求中超越自我，是追求人生的意义。

世界建立在荒诞之上，人是孤独的、是苦闷是对自身持有怀疑的。人都承受着虚无带来的痛苦与煎熬。正是如此，人才会对追求个性、追求自由的浪漫主义所感染。存在主义从个人与世界的关系的角度将荒诞界定为个人在生存中的一种生存体验，它既是颠覆理性后的孤独感，也是对未知可能性的恐惧和人生无意义的绝望，而正是在这片荒土上构筑出了浪漫主义。

02 乌鸦

本系列纯属偏见

本着任何的观点本质上都是不同学识的人在不同的世界观下对同一件事情的偏见，所以任何的偏见都在某种意义上属于对的。以此为本系列大前提而进行的任何偏见言论，都应当被尊重。

本系列大多数为碎碎念，想到什么写什么，并且我不对其负责。

以下内容为偏见：若要证明“所有的乌鸦都是黑的”，需要做的不是去找出成百上千只黑色的乌鸦，而是千方百计的去证明“有乌鸦不是黑色的”。

如果从归纳法的角度出发，要证明“所有的乌鸦都是黑色的”这个观点，自然是找成百上千只黑色的乌鸦。每一次的观察过后都会加深对“所有的乌鸦都是黑色的”这个观点的可信度。

但同时在逻辑上我们可以这么思考：乌鸦都是黑色的，所以不是黑色的东西就肯定不是乌鸦。因为“所有的乌鸦都是黑色的”这个命题的逆否命题就是“所有不是黑色的东西都不是乌鸦”。
而在这层逻辑下，每看到一个不是黑色的物体，它不是乌鸦，按照上面归纳法的逻辑自然等于增加了“所有的乌鸦都是黑色的”命题的可信性。

而这层逻辑下就等同于在证明“有乌鸦不是黑色的”。
结论1：如果要证明一个观点，要做的不是去找能证明这个观点的论据，而是去证明这个观点是错误的。正如福尔摩斯所说：“当你排除一切不可能的情况，剩下的，不管多难以置信，那都是事实”。
结论2：归纳法反直觉，应该少用归纳法来论证。
理论应该结合实际，从现实出发就能更好的理解这种逻辑。案例：某知名手机厂商在广告中暗讽友商产品。分析：其实这就是典型的“若要证明A，我去证明非A，因此可以证明A”，在逻辑归纳上是成立的，虽然有点反直觉。

该厂商思维：我要证明我的产品是好的，在逻辑上我只要证明不是我们的产品都是差的，就等同于证明了我们的产品是好的。

虽然某种意义上说他的思维没错，但如果按照本偏见来推导，正确的步骤应该是寻找证明我们的产品是差的，以此来证明我们的产品是好的。
不过在营销上没有人会去试图揭露自己产品的弊端。那在营销上怎么证明自己的产品是好的呢？于是就是证明自己产品是好的的逆否命题，也就是别人的都不行。
或许这是营销里的悖论，也是市场必将背负的宿命吧。

01 自我欺骗

本系列纯属偏见

本着任何的观点本质上都是不同学识的人在不同的世界观下对同一件事情的偏见，所以任何的偏见都在某种意义上属于对的。以此为本系列大前提而进行的任何偏见言论，都应当被尊重。

本系列大多数为碎碎念，想到什么写什么，并且我不对其负责。

以下内容为偏见​：

人大致分两种状态，一种是从自我欺骗进化到欺骗他人，一种是欺骗他人升华到自我欺骗。前者是自卑的人的活法，后者是自信的人的活法，普通人可能就在两者之间不断徘徊。

自我欺骗是一种很常见的心理状态，可以被多种原因激发​。通过欺骗自己使得自己接触不到真实的信息，《自私的基因》一书中有一种观点，自我欺骗是为了更好的欺骗其他个体，只要骗人的得益大于自我欺骗本身（接触不到真实的信息），自我欺骗就自然不会被生物进化论所淘汰。

但还有一种情况，便是自我欺骗的个体并没有意识到自己在进行自我欺骗。掩耳盗铃、一叶障目都是经典的自我欺骗的案例，并企图进化为欺骗他人。在这些成语典故里，自我欺骗者就没有意识到自己在进行自我欺骗，而是对事实有不同观点。但这种能否定义为自我欺骗，因为语义上自我欺骗意味着自己知道真实的信息是什么，并且坚定的相信​错误的信息。

在认知偏差的研究中，人们会将自己不想知道的信息过滤掉，比如你肯定想不起昨天收了个什么垃圾短信；在传播学里也有类似的观点：受众普遍会倾向于接受自己已经接受过的观点，这就导致了有不同党性的媒体有不同的受众，让民主党的受众去看共和党的报纸肯定会当场撕了那张报纸。

由此大致可以推出：自我欺骗可以帮助个体抵御负面信息，让个体在生产工作时不被负面反馈影响。有研究表明，自欺还可以增强个体的控制感 并使个体维持较高的道德标准, 促使利他行为的出现。但同时，自我欺骗妨碍个体正确认识自我，改正错误。自我欺骗者大多焦虑敏感水平较低，倾向于选择较少的选择来解决问题，从而减少努力在​生活中的作用。

结论：自我欺骗能给提升个体的自信，完成本来极有可能无法完成的事情​；但也有可能带来​危险却不自知。​
